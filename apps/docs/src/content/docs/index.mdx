## 設計の考え方

<CardGrid>
  <Card title="静的ファイルとして残る" icon="document">
    音声・メタデータ・RSS フィードはすべて R2 上の静的ファイルです。データベースも状態も持ちません。ファイルが存在する限り、配信は続きます。
  </Card>
  <Card title="使った分だけのコスト" icon="setting">
    Cloudflare Workers と R2 の従量課金です。配信がなければコストはほぼゼロです。急激なトラフィック増にも追加作業なしで対応できます。
  </Card>
  <Card title="データはあなたの管理下に" icon="approve-check">
    すべてのデータはあなたの R2 バケットに保存されます。プラットフォームに預けません。移行・バックアップは単純なファイル操作で完結します。
  </Card>
  <Card title="実績ある技術だけを使う" icon="puzzle">
    Workers、Pages、R2 — Cloudflare が世界規模で運用するインフラのみを使用しています。独自の複雑な仕組みは持ち込みません。
  </Card>
</CardGrid>

## システム構成

| コンポーネント | 技術 | 役割 |
|---|---|---|
| API | Cloudflare Workers + Hono | エピソード管理・アップロード処理 |
| 公開サイト | Astro SSG | リスナー向け静的サイト |
| 管理画面 | React + Vite | エピソード・設定の管理 |
| ストレージ | Cloudflare R2 | 音声・メタデータ・RSS フィード |
| 認証 | Cloudflare Access | ゼロトラスト認証 |
